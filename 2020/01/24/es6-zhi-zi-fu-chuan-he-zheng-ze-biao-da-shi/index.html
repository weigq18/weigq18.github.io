ES6之字符串和正则表达式

ES6笔记
发布日期:   2020-01-24

更好的Unicode支持

早期，由于存储空间宝贵，Unicode使用16进制来存储文字。我们将一个16位的二进制编码叫做码元(Code Unit).

2^16 = 65536 对于全世界的存储来说 是不够的。

后来，由于技术的发展，Unicode对文字编码进行扩展，将某些文字扩展到了32位(占用两个码元)，并且，讲某个文字对应的二进制数字叫做码点（Code Point）。

2^32 = 429296729 目前够用 有的文字一个码点需要占用两个码元 有的文字的一个码点只占用一个码元

const text = "𠮷";  //字符串是根据码元来取的  这个字占用了两个码元（32位）

console.log("字符串长度：", text.length); // 2

//正则是用码元来匹配的 不是用码点来匹配的  所以占两个码元
//正则中^代表开头 .代表容易单个字符串 $代表结尾
//为false 因为吉这个字是由一个码点 两个码元组成 而正则是根据码元来匹配 但是码元有两个。
console.log("使用正则测试：", /^.$/.test(text)) 

// 以前的方法
console.log("得到第一个码元", text.charCodeAt(0).toString(16))
console.log("得到第二个码元", text.charCodeAt(1).toString(16))

//ES6为了解决这个困扰，为字符串提供了方法： codePointAt, 根据字符串码元的位置得到

//𠮷: \ud842\udfb7
console.log("得到第一个码点：", text.codePointAt(0));
//𠮷 这个字可以拆分为 \ud842 和 \udfb7  如果算第二个码点 就算\udfb7 
console.log("得到第二个码点：", text.codePointAt(1));

// 判断文字是32位还是16位
function is32bit(char, i) {
    // 判断这个码点是否大于16位的最大值 如果大于 则其是32位
    return char.codePointAt(0) > 0xffff;
}


// 得到字符串码点的真实长度
function getLengthOfCodePoint(str) {
    var len = 0;
    for(let i = 0; i < str.length; i++) {
        // i在索引码元
        if(is32bit(str, i)) {
            //当前字符串在i这个位置，占用两个码元
            i++;
        }
        len++;
    }
    return len;
}
console.log("𠮷是否是32位的", is32bit("𠮷"))


//同时，ES6为正则添加了一个flag：u，如果添加了该配置，则匹配时，使用码点匹配
console.log("使用正则测试：", /^.$/u.test(text))

更多的字符串API

以下均为字符串的实例方法（原型上的方法）

  • includes 判断一个字符串中是否包含指定的子字符串
  • startsWith 判断字符串中是否以指定的字符串开始
  • endsWith 判断字符串中是否以指定的字符串结尾
  • repeat 将字符串重复指定的次数，然后返回一个新字符串
//判断是否包含狠字
const text = "是个狠人";
//从字符串下标为0的开始查找
const result = text.includes('狠');
//从字符串下标为3的开始查找
const result = text.includes('狠', 3);

console.log("是否包含“狠”", text.includes("狠"));
console.log("是否以“是”开头", text.startsWith("是"));
console.log("是否以“人”结尾", text.endsWith("人"));
console.log("重复4次", text.repeat(4));

正则中的粘连标记

标记名：y

含义：匹配时，完全按照正则对象中的lastIndex位置开始匹配，并且匹配的位置必须在lastIndex位置

const text = "Hello World!!!";
const reg = /W\w+/; //以W字符开头+任意的单词字符一次或多次
console.log(reg.text(text));  //true

// 如果加上y 表示粘连关系 根据lastIndex来判断。因为lastIndex为3，匹配所以为true lastIndex默认为0
const regY = /W\w+/y;
regY.lastIndex = 3;
console.log("reg.lastIndex:", reg.lastIndex); //true
console.log(reg.test(text));

模板字符串

ES6之前处理字符串繁琐的两个方面

  1. 多行字符串
  2. 字符串拼接

ES6中，提供了模板字符串的书写，可以非常方便的换行和拼接，要做的仅仅是将字符串的开始或结尾改为`符号

如果要在字符串中拼接js表达式，只需要在模板字符串中使用${}

// 以前处理方式 要用\n来换行 或者加斜杆换行
var text = "好好学习\n天天向上"
var text1 = "好好学习\
                    天天向上"
var text2 = ['好好学习', '天天向上'].join("\n")

// 模板字符串
var newText =  `好好学习
天天向上`

// 模板字符串拼接

var condition = '废寝忘食'
var newText =  `好好学习
${condition}天天向上,
表达式可以是任何有意义的数据${1 + 3 * 2 / 0.5}
表达式是可以嵌套的${`表达式中的模板字符串`}`

模板字符串的标记

在模板字符串书写之前，可以加上标记：

标记名`模板字符串`
var newTextTag =  myTag`好好学习
${condition}天天向上, ${condition}。`

//相当于  newTextTag = myTag(['好好学习', '天天向上', '。'], '废寝忘食', '废寝忘食')

//实现模板字符串拼接
function myTag(parts, arg1, arg2) {
    const values = Array.prototype.slice.apply(arguments).slice(1);
    let str = '';
    for(let i = 0; i < values.length; i++) {
        str += parts[i] + values[i];
        if(i === values.length - 1) {
            str += parts[i + 1];
        }
    }
    return str;
}

console.log(newTextTag)  //好好学习废寝忘食天天向上，废寝忘食。

//String.raw 把转义字符变成字符串
var text = String.raw`abc\nbcd`;  //abc\nbcd

标记是一个函数，函数参数如下：

  1. 参数1： 被插值分割的字符串数组
  2. 后续参数：所有的插值
